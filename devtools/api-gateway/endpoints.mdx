**Endpoints** são as rotas HTTP que o seu gateway expõe ao mundo. Cada endpoint combina um método, um caminho e uma integração que define de onde vem a resposta.

## Anatomia de um Endpoint

| Campo | Descrição |
|-------|-----------|
| **Método** | Verbo HTTP — GET, POST, PUT, PATCH ou DELETE |
| **URN** | Caminho da rota, ex: `/usuarios/:id` |
| **Alias** | Nome interno único para identificar o endpoint |
| **Integração** | O que executa quando a rota é chamada |
| **Pasta** | Agrupamento opcional para organizar os endpoints na interface |

<Note>
  Alias e a combinação URN + Método devem ser únicos dentro de uma release.
</Note>

## Tipos de integração

Ao criar um endpoint, você escolhe como ele vai responder às requisições:

  <Card title="Conector" icon="plug">

  <Card title="Repositório" icon="code-branch">

## Editando e removendo

- **Editar**: clique no endpoint na barra lateral e altere os campos desejados
- **Remover**: use a opção Apagar no menu de ações do endpoint

### Lixeira

Endpoints removidos ficam disponíveis na **Lixeira** e podem ser restaurados enquanto a release estiver no status `BUILDING`. Após o encerramento da release, a lixeira não está mais disponível.

---

## Salvar vs Publicar

| Ação | O que faz |
|------|-----------|
| **Salvar** | Persiste as alterações na release — ainda não reflete no Gateway Runtime |
| **Publicar** | Sincroniza as configurações com o Gateway Runtime e torna a API acessível |
