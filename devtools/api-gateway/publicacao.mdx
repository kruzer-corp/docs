**Publicar** é o ato de sincronizar as configurações da sua release com o Gateway Runtime — tornando os endpoints acessíveis para os consumidores. Sem publicar, as alterações existem apenas na plataforma de gestão e não refletem na API em execução.

## O que acontece ao publicar

<Steps>
  <Step title="Configurações sincronizadas">
    Endpoints, autenticação, políticas e acessos do ambiente selecionado são enviados ao Gateway Runtime em tempo real.
  </Step>
  <Step title="API disponível">
    Os endpoints ativos no ambiente passam a responder requisições na URL base configurada.
  </Step>
  <Step title="Release encerrada (opcional)">
    Se você optar por encerrar a release atual, ela muda para o status `CLOSED` e uma nova release é criada automaticamente com todos os endpoints copiados.
  </Step>
</Steps>

---

## Formulário de publicação

| Campo | Descrição |
|-------|-----------|
| **Destino** | Ambiente para o qual a release será publicada |
| **Comentário** | Descrição obrigatória sobre o que está sendo publicado |
| **Encerrar release atual?** | Sim/Não — se Sim, a release atual é fechada e uma nova é criada |
| **Nome da nova release** | Obrigatório caso opte por encerrar — nome livre (ex: `2.0`, `v2-beta`) |

<Tip>
  Use o comentário de publicação para registrar o contexto da mudança — ele fica no histórico de lançamentos e facilita auditorias futuras.
</Tip>

## Encerrar ou não a release?

<CardGroup cols={2}>
  <Card title="Manter a release aberta" icon="pen">
    A release continua no status `BUILDING`. Você pode seguir editando endpoints e publicar novamente no futuro.

    **Indicado para:** ajustes incrementais sem necessidade de versionar.
  </Card>
  <Card title="Encerrar a release" icon="tag">
    A release atual vai para `CLOSED` e uma nova é criada com os mesmos endpoints.

    **Indicado para:** marcos de versão, breaking changes ou quando um contrato precisa ser preservado.
  </Card>
</CardGroup>

<Warning>
  Ao encerrar uma release, os IDs dos endpoints mudam na nova release. Policies vinculadas a endpoints precisam ser reconfiguradas após a publicação.
</Warning>
