**Ambientes** são instâncias de execução do seu gateway — cada um referenciando uma Release específica e com sua própria URL base, autenticação, endpoints ativos e políticas. O Ambiente determina **como** a API é acessada; a Release determina **quais endpoints** estão disponíveis para exposição.

Isso permite que Dev aponte para a versão mais recente em desenvolvimento, enquanto Produção continua expondo uma versão estável — de forma totalmente independente.

## O que define um Ambiente

| Campo | Descrição |
|-------|-----------|
| **Nome** | Identificador do ambiente (ex: Development, QA, Production) |
| **Release** | Release referenciada — define quais endpoints esse ambiente pode expor |
| **URL base** | Host pelo qual a API será acessada externamente |
| **Autenticação** | Método de autenticação exigido para consumir a API |
| **Status** | Ativo ou Inativo |

---

## Abas do Ambiente

Cada ambiente é gerenciado por três abas:

### Autenticação

Define como os consumidores da API devem se autenticar. Os tipos disponíveis são:

<AccordionGroup>
  <Accordion title="Sem autenticação">
    A API é pública — qualquer cliente pode fazer requisições sem credenciais.

    Indicado para endpoints de uso aberto ou em ambientes de desenvolvimento local.
  </Accordion>

  <Accordion title="Basic Auth">
    Autenticação com usuário e senha codificados em Base64.

    ```
    Authorization: Basic dXN1YXJpbzpzZW5oYQ==
    ```

    A plataforma gera as credenciais automaticamente ao criar um Acesso.
  </Accordion>

  <Accordion title="API Key">
    Chave única enviada no header da requisição.

    ```
    X-API-Key: sua-chave-gerada-pela-plataforma
    ```

    Simples de implementar e adequado para integrações B2B.
  </Accordion>

  <Accordion title="OAuth2">
    Fluxo completo de autenticação com emissão e renovação de token.

    Recomendado para APIs expostas a parceiros externos que exigem maior segurança.
  </Accordion>
</AccordionGroup>

<Warning>
  Ao alterar o tipo de autenticação de um ambiente, **todos os Acessos existentes são removidos**. Novos acessos precisam ser criados com o novo tipo.
</Warning>

### Gerenciamento de Endpoints

Lista todos os endpoints da release vinculada ao ambiente. Aqui você decide quais estarão ativos ou inativos neste ambiente específico.

- **Ativar/desativar individualmente** — clique no toggle de cada endpoint
- **Ações em massa** — ative todos, desative ou exclua os selecionados de uma vez
- **Filtros** — pesquise por rota, método HTTP ou status

<Note>
  Um endpoint inativo não é acessível pelos consumidores, mesmo que exista na release. Isso permite que você controle gradualmente o que cada ambiente expõe.
</Note>

### Policies

Lista e gerencia as políticas de segurança e performance vinculadas a este ambiente. Veja [Policies](/api-gateway/policies) para detalhes sobre cada tipo disponível.
