**Consumers** são serviços que ficam escutando filas de mensageria e executam uma automação a cada mensagem recebida. Enquanto os Triggers executam por agendamento ou webhook, os Consumers respondem a **eventos de fila** — ideais para integrações assíncronas e processamento contínuo de mensagens.

## Como funciona

Ao receber uma mensagem na fila, o Consumer instancia a automação configurada, passando o conteúdo da mensagem como input. A execução acontece de forma isolada para cada mensagem, com rastreabilidade individual via logs.

## Brokers suportados

A plataforma suporta diferentes brokers de mensageria. A configuração varia conforme o broker escolhido — cada um possui seus próprios parâmetros de conexão.

Abaixo, exemplos de configuração para alguns dos brokers disponíveis:

<AccordionGroup>
  <Accordion title="Apache Kafka">
    Broker de streaming distribuído de alta vazão, amplamente usado para pipelines de eventos em tempo real.

    | Parâmetro | Descrição |
    |-----------|-----------|
    | Brokers | Lista de endereços dos brokers (ex: `kafka:9092`) |
    | Client ID | Identificador do cliente na conexão |
    | Group ID | Grupo de consumidores para balanceamento |
    | Topic | Tópico que o Consumer irá escutar |
    | Dead Letter Topic | Tópico para mensagens com falha (opcional) |
  </Accordion>

  <Accordion title="IBM MQ">
    Broker corporativo da IBM, comum em integrações com sistemas legados e ambientes financeiros/industriais.

    | Parâmetro | Descrição |
    |-----------|-----------|
    | Host | Endereço do servidor IBM MQ |
    | Porta | Porta de conexão |
    | Channel | Canal de comunicação |
    | Queue Manager | Nome do gerenciador de filas |
    | Queue Name | Nome da fila a ser consumida |
  </Accordion>
</AccordionGroup>

<Note>
  A plataforma está em constante evolução com suporte a novos brokers. Consulte a documentação ou entre em contato com o time Kruzer para verificar as integrações disponíveis.
</Note>

## Configuração geral

Independente do broker escolhido, todo Consumer compartilha os mesmos campos base:

| Campo | Descrição |
|-------|-----------|
| **Nome** | Identificador do Consumer na plataforma |
| **Broker** | Tipo de mensageria utilizado |
| **Tenant** | Tenant ao qual o Consumer está vinculado |
| **Repositório** | Repositório que contém o código da automação |
| **Branch** | Branch do repositório |
| **Arquivo** | Arquivo de automação que será executado a cada mensagem |
| **Logger name** | Nome do serviço de log para rastreabilidade das execuções |
| **Status** | Ativo ou Inativo |

<Warning>
  Um Consumer **ativo não pode ser editado**. Para alterar qualquer configuração, desative-o primeiro, faça as alterações e reative.
</Warning>
