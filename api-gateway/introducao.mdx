O **API Gateway** do Kruzer DevTools permite que times de desenvolvimento construam e disponibilizem APIs ao mercado de forma estruturada. Com ele, você centraliza o controle de acesso, versiona contratos de API e configura ambientes independentes — tudo sem precisar gerenciar infraestrutura de gateway por conta própria.

## Como o Gateway se encaixa no DevTools

O API Gateway consome os recursos que você já configurou na plataforma. Os endpoints do gateway apontam diretamente para **conectores** ou **arquivos de automação** dos seus repositórios — sem necessidade de duplicar lógica.

```
Cliente externo
      ↓
 API Gateway  ←  Políticas de segurança (Rate Limit, Quota, Cache, Inbound Rules)
      ↓
Conector iPaaS  ou  Arquivo de automação (repositório)
      ↓
  Sistemas integrados (APIs, bancos, filas...)
```
