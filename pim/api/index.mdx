## Autenticação

Toda requisição à API exige autenticação via **JWT Bearer Token**. Para integrações sistêmicas, recomenda-se usar uma **conta de serviço** dedicada (ver [Identidade e Acesso](/plataforma-kruzer/iam-autenticacao)).

```http
GET /products HTTP/1.1
Host: api.kruzer.io
Authorization: Bearer <seu-token-jwt>
```

<Warning>
  Tokens têm validade limitada. Para integrações de longo prazo, implemente o fluxo de renovação descrito em [Autenticação](/plataforma-kruzer/iam-autenticacao).
</Warning>

### Formato

- **Request/response**: JSON (`Content-Type: application/json`)
- **Datas**: ISO 8601 (`2026-04-30T15:00:00Z`)
- **IDs**: strings opacas — não tente interpretar o formato

### Paginação

Listagens suportam paginação. Os parâmetros e formato exatos estão documentados em cada endpoint na seção abaixo.

### Erros

Códigos HTTP padrão. O corpo da resposta de erro segue o formato:

```json
{
  "error": "ValidationError",
  "message": "Descrição legível do erro",
  "details": { }
}
```

### Multi-idioma

Endpoints que retornam campos multi-idioma (nome, descrição) respeitam o header `Accept-Language` quando enviado, ou retornam o idioma padrão do catálogo.

## Endpoints

A referência completa de endpoints é **gerada automaticamente** a partir dos controllers do PIM e disponibilizada nesta seção. Os endpoints aparecem agrupados por domínio (Produtos, Categorias, Importações, Workflows, etc.) no menu lateral.

Cada endpoint exibe:

- **Método HTTP e rota**
- **Parâmetros** (path, query)
- **Request body** (quando aplicável)
- **Códigos de resposta** e formato

**Total atual**: 150 endpoints distribuídos em 29 domínios.

<Note>
  Os schemas de request/response estão na versão inicial — campos detalhados serão enriquecidos progressivamente. Para o modelo de dados completo de cada entidade, consulte [Conceitos do PIM](/pim/conceitos/produtos-e-skus).
</Note>
