O **Produto** é a entidade central do PIM — toda a operação gira em torno dele. O PIM trata o produto como um objeto rico, multi-idioma, com atributos dinâmicos, mídia vinculada e variações (SKUs).

## Produto vs. SKU

<CardGroup cols={2}>
  <Card title="Produto" icon="box">
    A entidade comercial conceitual: "Tênis Modelo X". Carrega informações compartilhadas: descrição, marca, família, categoria, mídia.
  </Card>
  <Card title="SKU" icon="barcode">
    A variação vendável: "Tênis Modelo X — Tamanho 42, Cor Preta". Carrega o que diferencia a variação: código de barras, dimensões, preço, estoque.
  </Card>
</CardGroup>

Um produto pode ter **uma ou várias SKUs**. Produtos simples (sem variação) têm apenas uma SKU. Produtos com variações de tamanho/cor/voltagem têm uma SKU por combinação.

## O Que Compõe um Produto

Um produto no PIM carrega:

| Categoria | Conteúdo |
|-----------|----------|
| **Identificação** | Código, nome, descrição (todos suportam multi-idioma) |
| **Classificação** | Família, categoria(s), marca |
| **Atributos** | Campos dinâmicos definidos pela [família](/pim/conceitos/familias-e-atributos) |
| **Mídia** | Imagens, vídeos, documentos |
| **Comerciais** | SKUs, preços, estoques |
| **Relacionamentos** | Produtos relacionados, associações de catálogo |
| **Workflow** | Status no [workflow de aprovação](/pim/conceitos/workflow-de-aprovacao) |
| **Auditoria** | Quem criou/alterou, quando, snapshots |

---

## Multi-idioma

Campos textuais (nome, descrição, valores de atributo do tipo texto) suportam múltiplos idiomas. Cada catálogo define quais idiomas são habilitados (ver Master Data → Idiomas).

A interface permite preencher e visualizar os mesmos produtos em diferentes idiomas. A API expõe os valores por idioma e respeita o idioma corrente do usuário.

---

## Atributos Dinâmicos

Diferente de um banco de dados tradicional, o PIM **não tem colunas fixas** para informações de produto. Os atributos disponíveis são definidos pela [família](/pim/conceitos/familias-e-atributos) — cada família carrega seu próprio conjunto.

Isso significa que adicionar um novo atributo não exige migração: basta criá-lo na família e ele aparece nos produtos correspondentes.
