A Kruzer é uma **plataforma modular**: cada produto resolve um conjunto específico de problemas, mas todos compartilham a mesma camada de identidade, organização e governança. Você pode adotar um módulo isoladamente ou combinar vários conforme a necessidade do seu negócio.

## O Que é Compartilhado

Independentemente do módulo que você utiliza, alguns serviços são **transversais** — atendem todos os produtos da Kruzer:

<CardGroup cols={2}>
  <Card title="Identidade (IAM)" icon="user-shield" href="/plataforma-kruzer/iam-autenticacao">
    Autenticação única com JWT, contas de serviço e SSO corporativo
  </Card>
  <Card title="Permissões (RBAC)" icon="key" href="/plataforma-kruzer/iam-permissoes">
    Controle de acesso baseado em papéis e grupos
  </Card>
</CardGroup>

Isso significa que **um mesmo usuário** acessa DevTools e PIM com a mesma credencial; **uma mesma conta de serviço** pode integrar sistemas externos com qualquer módulo; **as permissões** são geridas no mesmo lugar.

---

## Como os Módulos Conversam

Os módulos da Kruzer foram desenhados para funcionar de forma independente, mas se beneficiam quando combinados:

- **PIM + DevTools**: o iPaaS pode sincronizar produtos do PIM com sistemas externos (ERPs, marketplaces, e-commerces) usando os webhooks do PIM como gatilho.
- **API Gateway + PIM**: o gateway pode expor parte do catálogo PIM como uma API pública para parceiros, com versionamento, rate limit e quotas.
- **OMS + PIM**: pedidos do OMS consultam o catálogo do PIM para enriquecer informações de produto.
